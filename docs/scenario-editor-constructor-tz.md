# ТЗ: Конструктор сценариев `/builder`

Статус: актуализировано под текущую реализацию.

## 1. Цель

Дать пользователю визуальный редактор диалогового тренажера:

- строить граф нод и связей;
- редактировать контент в правой панели;
- запускать preview;
- публиковать валидный сценарий.

## 2. Экран и компоновка

- верхний header (`Preview`, `Publish`, контекст проекта);
- левый toolbar инструментов;
- центральный canvas с zoom/pan;
- правая панель настроек выбранной ноды (показывается только при выборе).

## 3. Типы нод

### Start

- ровно одна нода на сценарий;
- нельзя удалить;
- без входящих связей;
- от Start должен быть один первый Message.

### Message

- реплика NPC/системы;
- поля: `speakerType`, `speakerName`, `text`, `emotionState`, `characterAsset`, `backgroundAsset`;
- должен вести как минимум к одному Response или End.

### Response

- вариант ответа ученика;
- поля: `responseText`, `scoreDelta`, `feedbackType`, `hintText`, `nextRef`;
- должен иметь входящую и исходящую связь.

### End

- финал ветки;
- поля: `endingType`, `title`, `description`;
- исходящих связей нет.

## 4. UX правила

- drag and drop нод по canvas;
- соединение нод через коннекторы;
- удаление ноды удаляет связанные edges;
- zoom `25%..200%` без “разрыва” линий;
- dirty-state (несохраненные изменения) + предупреждение при уходе.

## 5. Валидация перед публикацией

Blocking:

- нет/несколько Start;
- Start не связан с первым Message;
- есть недостижимые ноды;
- отсутствуют End-ноды;
- не заполнены обязательные поля;
- Response без следующего шага.

Warnings:

- слишком длинные тексты;
- слишком большое число вариантов ответа в одном шаге.

## 6. AI-помощник в конструкторе

- кнопка `✨` рядом со Start;
- открывает правый AI-дровер;
- пользователь задает описание;
- по клику “Сгенерировать сценарий” backend строит граф.

Ограничение доступа:

- доступно только для тарифов `pro` и `enterprise`.

## 7. API контракты редактора

- `GET /api/v1/builder/dialogs/:id/editor` — получить graph;
- `PUT /api/v1/builder/dialogs/:id/editor` — сохранить graph;
- `POST /api/v1/builder/dialogs/:id/ai/generate` — AI-генерация;
- `POST /api/v1/builder/dialogs/:id/publish` — публикация;
- `POST /api/v1/builder/dialogs/:id/preview/link` — временный preview token.

## 8. Acceptance criteria

1. после создания проекта есть автопереход в `/builder/dialog/:id`;
2. все 4 типа нод редактируются и сохраняются;
3. валидация корректно блокирует publish на ошибках;
4. preview открывается и воспроизводит текущий graph;
5. AI генерирует ветвящийся, а не линейный сценарий;
6. права доступа соблюдаются по роли и тарифу.
