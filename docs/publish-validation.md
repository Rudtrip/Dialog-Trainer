# Publish Validation Guide

Файл схемы: `schemas/scenario.schema.json`

## 1. Структурная проверка

Перед публикацией backend валидирует:

- корректность структуры `graph` (`nodes`, `edges`);
- обязательные поля нод по их типу;
- корректность ссылок между нодами;
- отсутствие неизвестных `node.type`.

## 2. Семантическая проверка графа

Блокирующие ошибки:

- нет `Start`;
- больше одной `Start`;
- у `Start` нет исходящей связи;
- из `Start` допустим только один первый `Message`;
- есть недостижимые ноды;
- нет достижимой финальной ветки;
- `Response` без входящей или исходящей связи;
- `End` с исходящей связью;
- ссылки нод указывают на несуществующие id.

## 3. Контентные проверки

- `Message.text` и `Response.responseText` не пустые;
- длины строк в допустимых пределах;
- корректный `scoreDelta` для `Response`;
- корректные `emotionState` для `Message` (`neutral|happy|concerned|angry`).

## 4. Проверка лимитов и доступа

- пользователь должен иметь доступ к workspace;
- создание/дублирование симулятора ограничено тарифом (`simulator_limit`);
- AI-генерация доступна только для `pro` и `enterprise`.

## 5. AI-ветвление (важно)

Для AI-сценариев backend нормализует структуру и требует ветвление:

- ответы должны вести в разные ветки, а не в одно сообщение;
- у `Response` должна быть явная маршрутизация (`nextType/nextRef`);
- если модель вернула неполную структуру, применяется безопасная нормализация.

## 6. Поведение при ошибках

- `blocking errors` -> публикация запрещена;
- `warnings` -> публикация разрешена, но выводится предупреждение.

## 7. Рекомендуемый pipeline

1. JSON schema validation.
2. Graph semantic validation.
3. Scoring/access/tariff/media checks.
4. Формирование immutable snapshot публикации.
