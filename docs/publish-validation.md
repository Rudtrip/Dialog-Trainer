# Publish Validation Guide

Файл схемы: `schemas/scenario.schema.json`

## 1. Что проверяется структурно

- обязательные блоки сценария;
- типы полей и диапазоны;
- формат идентификаторов;
- корректность ссылок (`nextStepId`/`endingId` в пределах JSON-модели).

## 2. Что проверяется семантически (backend)

- есть ровно один `Start`;
- из `Start` есть только один первый `Message`;
- нет недостижимых нод;
- из стартовой точки достижима хотя бы одна `End` нода;
- `Response` имеет корректную исходящую связь;
- `passThreshold <= maxScore` (если задан порог);
- проверка тарифных ограничений (для действий создания/дублирования);
- media refs указывают на валидные ассеты.

## 3. AI-сценарии

Для AI-генерации дополнительно:

- ответы должны иметь явную маршрутизацию (`nextType`, `nextRef`);
- первая NPC-реплика должна реально ветвиться, а не вести всеми ответами в одно сообщение;
- если AI вернул неполную структуру, backend применяет безопасную нормализацию.

## 4. Поведение при ошибках

- blocking errors: publish запрещен;
- warnings: publish разрешен, но пользователь видит предупреждения.

## 5. Рекомендуемый pipeline

1. Структурная проверка JSON.
2. Семантическая проверка графа.
3. Проверка scoring/media/tariff constraints.
4. Формирование immutable snapshot публикации.
